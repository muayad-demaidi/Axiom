# AXIOM - Intelligent Data Analytics Platform

## Overview
AXIOM is a comprehensive, intelligent data analytics system, currently migrating from Streamlit to a unified Next.js and FastAPI application. Its core purpose is to simplify complex data analysis, offering capabilities such as automated data cleaning, statistical analysis, interactive visualizations, predictive analytics powered by AI, and professional report generation. The platform includes an AI chat assistant for conversational interactions and operates on a 60-day free trial model with email notifications and a support contact form. A sophisticated SEO/GEO automation agent and a dedicated marketing site drive organic discovery. The business vision is to provide an accessible yet powerful data analysis tool, enabling users to gain quick, professional insights without requiring deep technical expertise. The project aims for market potential by offering a robust solution for data-driven decision-making across various industries.

## User Preferences
- Language: Arabic (Levantine dialect) for communication
- No payment integration - all tiers freely accessible
- Professional, sophisticated design aesthetic
- Column types displayed in English

## System Architecture
The application is undergoing migration to a unified architecture consisting of a **Next.js 14 (App Router + React + TS + Tailwind)** frontend and a **FastAPI** backend. The system leverages **Pandas** and **NumPy** for data manipulation, **Recharts** for interactive visualizations (Plotly is retired), and **Scikit-learn** for machine learning models. **SQLAlchemy** with **PostgreSQL** manages user, project, dataset, chat session, and support message data. **OpenAI GPT** powers AI-driven analysis, recommendations, and conversational features. User authentication utilizes **JWT** and **bcrypt**.

### UI/UX Decisions
The design theme, "Data Noir," features a dark aesthetic with deep navy and AXIOM-blue accents (`#60A5FA` / `#3b82f6` / `#2563eb`). The UI uses Inter for body text, JetBrains Mono for code, and SF Pro as a fallback. A theme toggle persists user preference, and a custom `DataStreamBackground.tsx` component provides an animated Matrix-style background on the landing page. Glassmorphism cards and a subtle matrix rain background animation contribute to a professional user experience. The layout is desktop-first, with a maximum content width of 1320px. After login, the workspace follows a Julius.ai-style chat-first pattern: `/app` is a centered "What do you want to analyze today?" landing with a single rounded chat input (Attach data + Connectors + circular send), and a global left sidebar (`ProductSidebar.tsx`) shows **+ New chat**, recent **Chats** across all projects, recent **Projects**, and a **Workspace** group with Files and Data Connectors. Submitting from the landing calls `POST /api/chats/quick` to find-or-create a "Quick Chats" project + new session and redirects to `/app/project/{pid}?session={sid}&q={text}`, where `ChatPanel`'s `initialPrompt` auto-sends once. The projects index moved to `/app/projects`; project workspaces keep a slim inner rail for that project's own chats and datasets.

### Technical Implementations
- **Data Processing**: Features include an ordered list of toggleable data cleaning substeps (e.g., duplicate removal, missing value handling), descriptive statistics, correlations, and distribution analysis.
- **Visualizations**: Supports various chart types, including bar, scatter, box, pie, line charts, heatmaps, with custom SVG for box plots and CSS grid for heatmaps.
- **Predictive Analytics**: Implements linear models, trend analysis, categorical data analysis, ML prediction models (RandomForest/LinearRegression), and K-Means risk clustering.
- **User Management**: Includes email/password authentication, registration, detailed user profiles, and a 60-day free trial. Project management is central to the user workflow.
- **Admin Support inbox (Task #156)**: Admins (`users.is_admin == True`) get an in-app inbox at `/app/admin/support` listing every contact-form submission newest-first (name / email / message / created_at) with a "Mark handled" / "Reopen" toggle and an "only un-handled" filter. Backed by `GET /api/support/messages` and `PATCH /api/support/messages/{id}`, both gated by `backend.auth.get_current_admin` (401 anon, 403 non-admin). The handled flag reuses the existing `support_messages.is_read` column so no schema change was needed. `/api/auth/me` now also returns `is_admin` so the sidebar can conditionally render the "Admin → Support inbox" link.
- **AI-Powered Chat**: A project-aware AI assistant leveraging OpenAI GPT, supporting multi-conversation sessions and a Julius.ai-style landing chat. The system prompt embeds a CRISP-DM playbook (Understand → Identify → Plan → Result → Caveats), common-data-challenge guidance, and per-data-type methodology (sales/RFM, surveys/sentiment, e-commerce/funnel, telemetry/anomaly, sports/regression). Sessions are persisted under a per-project `chat_sessions` table; new chats auto-title from the first user message; ownership is enforced on every endpoint (verified cross-user 404s). New endpoints `GET /api/chats/recent` (cross-project recency) and `POST /api/chats/quick` (auto-create Quick Chats project + session) power the landing flow.
- **Guided → Expert handoff (Task #162)**: In Guided Mode the chat system prompt (`backend/chat.py`) instructs the assistant to answer the user's question in plain language and then append a final-line sentinel `[switch_to_expert] <≤90-char reason>` whenever the ask is genuinely beyond Guided (named algorithms, hyperparameter tuning, statistical tests by name, raw metric numbers, custom feature engineering / encoding, code generation, time-series diagnostics). `frontend/src/components/product/ChatPanel.tsx` strips the sentinel via `parseSwitchHandoff` and renders an inline "Open in Expert Mode" pill that calls `useMode().setMode("expert")` (per-project override when inside a project, user-level preference on the home page). The directive explicitly overrides the older `_apply_mode_directive` "do NOT offer to switch modes" line so the sentinel still fires in Guided. Evaluation prompts:
  - **Should trigger** the handoff: "Tune the XGBoost learning rate on my churn data", "Run a Ljung–Box test on the residuals", "Give me R², adjusted R² and AIC for the regression", "Write the Python code for the KMeans pipeline", "Fit ARIMA(2,1,2) and report residual autocorrelation", "Use SHAP values to explain the top features".
  - **Should NOT trigger** (Guided handles them): "Show me top 10 customers by revenue", "Plot revenue by month", "What's the average order value by region?", "Are there missing values?", "Cluster my customers into 3 groups", "Summarise this dataset".
- **Guided predictive flow (Task #212)**: A three-phase wizard inside the Predictions tab of the Artifact Drawer (Scanning → Questioning → Result), Arabic-first / RTL. New backend pair `backend/predict_guided.py` (router) + `backend/predict_guided_service.py` (ML+LLM) expose `POST /api/predict/guided/analyze` (profile + Arabic clarifying questions) and `POST /api/predict/guided/run` (Prophet for time-series with sklearn linear-trend fallback, sklearn LinearRegression / RandomForest for driver-based regression). A 0–100 confidence gauge blends four sub-scores (data volume / quality / signal strength / time coverage). GPT-4o (existing OpenAI client, isolated in `predict_guided_service._chat_completion` for monkeypatching) generates Arabic clarifying questions and an Arabic narrative — every numeric value is pre-formatted server-side and the model is instructed to quote it verbatim, never invent a number. Results are persisted via `models.save_chat_artifact(kind="prediction")` with the payload-discriminator `flow="guided"` so they coexist with the legacy `predict_column` PredictionCard. Frontend modules: `GuidedPredictionWizard.tsx` + `GuidedPredictionCard.tsx` (with inline `ConfidenceGauge`); `ArtifactDrawer.tsx` shows an "ابدأ التنبؤ" CTA in the Predictions tab whenever no guided artifact exists for the active dataset, and dispatches existing prediction artifacts to the right card based on `result.flow`. The wizard fires `axiom:guided-predict:state` window events so `DataContextBar.tsx` can swap its status pill to "جاري التنبؤ…" while a run is in flight. UI strings are Arabic + RTL; code/comments stay English. Tests in `tests/test_predict_guided.py` cover happy time-series + driver paths, small-sample refusal, deterministic question / narrative fallbacks when the LLM is offline, and the verbatim-number constraint on the narrative.
- **Conversational EDA workspace (Task #158)**: Inside a chat session AXIOM ships a Julius-style EDA flow. On dataset upload the chat shows a dataset preview card (`/api/datasets/{id}/preview` + `/profile` + `/insights` + `/suggestions`) with first ~10–20 rows in an interactive table (sort/search/ask-about-cell), an automatic profile, a Surprise insights ribbon, and 5–8 CRISP-DM-aligned suggestion chips. The chat itself uses NDJSON streaming (`POST /api/chat/stream`) with OpenAI tool-calling: `profile_dataset`, `make_chart`, `predict_column`, `cluster_dataset`. Each tool persists a `chat_artifacts` row (kind, params, result, summary, pinned) and emits `tool_started`/`tool_finished` events that auto-open a right-side artifact drawer (tabs Profile / Visualize / Predictions / Clusters) with skeleton loaders, what-if sliders on predictions, and pin/delete toggles. **Pinning semantics**: only `profile`/`insight` artifacts are auto-pinned; `chart`/`prediction`/`cluster` default to `pinned=False` and the user opts them into the report by clicking the pin. The Final Report side tab (`/app/project/{id}/report?session=…`) renders artifacts plus an LLM-synthesised executive summary + key findings + recommendations (`backend/insights.synthesize_session_insights`, cached as a `kind='synthesis'` ChatArtifact keyed by the contributing artifact-id set), deterministic ±10/±25 % what-if recommendation tables for every prediction (`what_if_recommendations`), and a literal **PDF mini-preview iframe** that fetches the same `POST /api/chats/{sid}/report.pdf` (ReportLab + Matplotlib, `ax.bxp` for precomputed box stats) and re-renders on every Pinned-only toggle. All section headings (UI + PDF) are bilingual English / Levantine Arabic. Strict access control: every dataset endpoint and every tool call uses `models.get_dataset_record_strict` (user-scoped, optionally project-scoped) so nothing leaks across users or projects; cross-user 404 verified end-to-end. The chat stream supports `AbortController` cancellation and the workspace clears pending tool skeletons on session change and stream end so drawer state cannot leak across chats. Smoke tests live in `tests/test_insights_synthesis.py`.
- **Power BI–style aggregation engine (Task #208)**: `backend/aggregation.py` is the single source of truth for measure aggregation across the product. It infers per-field metadata (role: dimension/measure/key/date · default aggregation: SUM for additive measures, AVG-with-warning for `*_pct`/`*_rate`, COUNT for non-numeric, NONE for IDs · format kind: number/integer/currency/percent/date/text · precision · label) and persists user overrides under `DatasetRecord.summary_stats["_axiom_field_meta"]`. `aggregate()` honours the engine and emits `warnings`, `grand_total`, and a `chart_suggestion`. A shared pre-flight `validate_request()` runs ahead of every aggregate call (visualize, pivot, dashboard, chat make_chart) so the same "you're summing a percentage / using a key as a row" warnings appear on every surface. `explain_cell()` returns the formula, filter summary, contributing-row count and a sample for any one number; `validate_join()` classifies cardinality (1:1 / 1:N / N:N) on a candidate join key and refuses silent fan-out. `backend/bi.py` exposes `/api/bi/{id}/field-meta` (GET/PATCH/DELETE), `/api/bi/{id}/modeling` (grain + fan-out detection), `/api/bi/pivot` (Rows / Columns / Values / Filters with subtotals + cross-tab + date grain), `/api/bi/{id}/dashboard` (GET auto-builds + persists with **page-level slicers** as `?date_from`/`date_to`/`slicer_column`/`slicer_values`, PUT saves the user's tile layout, DELETE resets), `/api/bi/explain` (per-cell explanation), `/api/bi/export/csv` (streams the same pivot/tile result as CSV) and `/api/bi/relationships/validate` (multi-table join safeguard). Auto-dashboards now publish a `slicers` array + `section` tags (executive / trend / segmentation / operational) so the dashboard page can render grouped sections + a date-range + categorical slicer that filter every tile. The visualize page bar/line branches, the chat assistant `make_chart` tool (TOOL_SCHEMA now accepts an `aggregation` enum), the pivot page and the dashboard page all route through the same engine. The pivot page (`/app/pivot`) supports per-row date-grain selectors, Top/Bottom-N, subtotal/grand-total/drop-nulls toggles, click-to-cross-filter on chart bars and table rows, an Explain modal on every cell/KPI, and one-click CSV export. The dashboard page (`/app/dashboard`) renders sectioned tiles with per-tile Explain / CSV / Drill-through (which navigates to `/app/pivot?row=…&measure=…&agg=…&filter=…` so the user lands on the matching cross-tab). The fields page (`/app/fields`) is the per-column role / aggregation / format / label override editor. The model page (`/app/model`) renders the same modeling-safeguards warnings before training. The visualize page surfaces the resolved Y label / aggregation / grand total / engine warnings underneath the rendered chart.
- **Data Connectors**: A `/app/connectors` catalog presents data sources supported through Replit (PostgreSQL, MySQL, MongoDB, Snowflake, BigQuery, Databricks, Google Sheets, Airtable, Notion, Stripe, HubSpot, Salesforce, Linear, GitHub, Slack, Google Analytics, CSV upload, REST), grouped by category, each opening a modal that explains how to wire it up.
- **Report Generation**: PDF reports are generated server-side using ReportLab, incorporating cover pages, data tables, statistical summaries, distribution histograms (Matplotlib), and AI-generated insights.
- **SEO/GEO Automation**: A Python agent generates GEO-optimized pages, refreshes content, and performs brand-mention checks using trending topics from external sources.
- **Deployment**: Configured for `autoscale` on Replit, building the Next.js frontend and running FastAPI with uvicorn.
