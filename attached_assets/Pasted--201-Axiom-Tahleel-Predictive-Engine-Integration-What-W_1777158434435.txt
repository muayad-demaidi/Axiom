#201 - Axiom "Tahleel" Predictive Engine Integration
What & Why
Integrate a specialized predictive service (codenamed "Tahleel") into the existing Predictions tab in the Artifact Drawer. This service uses a hybrid approach: Real ML (Prophet/scikit-learn) for calculation and Claude 3.5 Sonnet for Arabic business context and conditional logic. It aims to provide "If-Then" scenarios with confidence scores.

Done looks like

Predictions Tab UI: When clicking the "Predictions" tab, if no prediction exists, it shows a "Start Tahleel Analysis" button.

The Wizard (Arabic-first):

Phase 1 (Scanning): A loading state "جاري فحص البيانات..." while the backend scans columns for time-series and key drivers.

Phase 2 (The Questions): 3-5 smart Arabic questions appear one-by-one (Wizard style). These are generated by Claude based on data anomalies (e.g., "لاحظنا انخفاض في مايو، هل كان هناك إغلاق؟").

Phase 3 (Input): User answers using simple Arabic UI components (Sliders for numeric change, Yes/No, or Dropdowns).

The Prediction Card: A beautifully rendered Arabic card showing:

Context Summary: (Arabic narrative).

Conditional Result: "إذا قمت بـ [X] ← نتوقع [Y]" (Numbers from ML, Narrative from Claude).

Confidence Meter: A visual gauge (0-100%) with an honest Arabic explanation of the score.

Actionable Recommendations: 2-3 specific Arabic bullet points.

Backend Integration:

A new endpoint /api/predict/analyze that triggers Claude to generate questions.

A new endpoint /api/predict/run that takes user answers + data, runs Prophet (for time-series) or Linear Regression (for drivers), and sends results back to Claude for final Arabic formatting.

Strict Constraint: Claude never hallucinates the numbers. The ML model outputs the raw forecast, and Claude only "wraps" it in business Arabic.

Technical Implementation Details

ML Engine: Use prophet for any date-based data. Use scikit-learn (RandomForest/LinearRegression) for relationship-based data.

Claude Prompting:

Prompt 1 (Question Gen): "Given this data summary [JSON], identify 3-5 business uncertainties in Arabic to improve prediction."

Prompt 2 (Narrative Gen): "Combine these ML results [Numbers] with user inputs [Answers] to write a conditional prediction card in Arabic."

Confidence Score: Calculated based on: (Data volume + Missing values + Correlation strength + Time span).

Steps

Backend Service: Create predict_service.py to handle ML logic and Claude calls.

API Routes: Add the analysis and execution endpoints to the FastAPI backend.

Frontend Tab: Build the PredictionsTab.tsx inside the Artifact Drawer with a state machine (Idle → Scanning → Questioning → Result).

Arabic UI Components: Create the Wizard flow (one question at a time) and the final "Prediction Result" card with the confidence gauge.

Telemetry: Ensure the Data Context Bar shows "Analyzing..." while Tahleel is running.
