# 📋 برومبت شامل: نظام تنبؤ ذكي متعدد المشاريع مع سياق تعلم تراكمي

## 🎯 الهدف الرئيسي
بناء أداة تنبؤ (Predictive Analytics Tool) تتعلم من **جميع البيانات عبر مشاريع المستخدم**، وتسمح له بالاختيار بين:
- التنبؤ بناءً على **الداتا الحالية فقط**
- التنبؤ بناءً على **كل الداتا المتراكمة في المشروع/الحساب**

مع تطبيق أفضل ممارسات علم البيانات لتجنب الـ **Data Leakage** وضمان تنبؤات واقعية وقابلة للتطبيق.

---

## 🔐 القواعد الذهبية (غير قابلة للتفاوض)

### 1. منع تسريب البيانات (Data Leakage Prevention)
```
قبل أي تدريب نموذج، افحص كل عمود (Feature) واسأل:
✅ هل هذه المعلومة متاحة قبل لحظة اتخاذ القرار؟
✅ هل هي سبب وليس نتيجة للهدف؟
✅ هل ارتباطها مع الهدف > 0.8؟ (إذا نعم → راجعها بدقة)

إذا كان الجواب "لا" لأي سؤال → استبعد العمود تلقائياً مع تنبيه المستخدم:
"⚠️ العمود [اسم_العمود] قد يسبب تسريب بيانات لأنه مرتبط مباشرة بالنتيجة. 
تم استبعاده لضمان مصداقية التنبؤ."
```

### 2. الفصل الزمني للبيانات (Temporal Validation)
```
- إذا احتوت الداتا على طابع زمني → درّب على الفترة الأقدم واختبر على الأحدث
- لا تخلط البيانات عشوائياً (random shuffle) إذا كان الهدف تنبؤاً مستقبلياً
- اعرض للمستخدم: "تم التدريب على [تاريخ_بداية] إلى [تاريخ_نهاية]، والاختبار على [فترة_المستقبل]"
```

### 3. الشفافية في الأداء (Performance Transparency)
```
اعرض دائماً:
• R² (معامل التحديد)
• MAE / RMSE (متوسط الخطأ)
• عدد العينات المستخدمة في التدريب
• نطاق القيم المتوقع (مين/ماكس)
• فاصل الثقة (إن أمكن): "التنبؤ: 75 ± 5 درجات"
```

## ⚙️ المواصفات التقنية (Backend Logic)

### 1. تدريب النموذج (Model Training):
```python
from sklearn.linear_model import LinearRegression, Ridge
from sklearn.model_selection import cross_val_score, train_test_split
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.compose import ColumnTransformer
from sklearn.pipeline import Pipeline
import numpy as np

def train_predictor(data, target, features, group_col=None):
    # 1. تنظيف البيانات
    df = data.dropna(subset=[target] + features)
    
    # 2. فصل المتغيرات الرقمية والفئوية
    numeric_feats = [f for f in features if df[f].dtype in ['int64', 'float64']]
    categorical_feats = [f for f in features if f not in numeric_feats]
    
    # 3. معالجة مسبقة
    preprocessor = ColumnTransformer([
        ('num', StandardScaler(), numeric_feats),
        ('cat', OneHotEncoder(handle_unknown='ignore'), categorical_feats)
    ])
    
    # 4. بناء الباييبلاين
    model = Pipeline([
        ('prep', preprocessor),
        ('reg', Ridge(alpha=1.0))  # Ridge أفضل من Linear للتعميم
    ])
    
    # 5. تدريب مع Cross-Validation
    X, y = df[features], df[target]
    cv_scores = cross_val_score(model, X, y, cv=5, scoring='r2')
    
    model.fit(X, y)
    
    return {
        'model': model,
        'r2_mean': cv_scores.mean(),
        'mae': np.mean(np.abs(y - model.predict(X))),
        'feature_names': model.named_steps['prep'].get_feature_names_out(features),
        'coefficients': model.named_steps['reg'].coef_
    }
```

### 3. مقارنة المجموعات (Group Comparison):
```python
def compare_groups(model_packages, user_inputs, group_col, group_values):
    """
    يعيد تنبؤاً لكل قيمة فريدة في عمود المجموعة
    """
    results = []
    for val in group_values:
        # حقن قيمة المجموعة في المدخلات
        inputs_with_group = {**user_inputs, group_col: val}
        
        # إذا كان لكل مجموعة نموذج منفصل
        if val in model_packages:
            pred = predict_live(model_packages[val], inputs_with_group)
        else:
            # استخدام النموذج العام
            pred = predict_live(model_packages['global'], inputs_with_group)
        
        results.append({
            'group_value': val,
            'prediction': pred['prediction'],
            'ci': pred['ci_95']
        })
    
    return results
```

## 🔄 سيناريو المستخدم الكامل (User Journey)

1. **يرفع المستخدم داتا جديدة** ← النظام يفحص الأعمدة تلقائياً
2. **يختار الهدف والميزات** ← النظام يحذر من أي عمود مشبوه (Data Leakage)
3. **يحدد نطاق البيانات** ← (حالي / مشروع / حساب)
4. **يضغط "درّب النموذج"** ← يظهر شريط تقدم + مقاييس الجودة
5. **يبدأ اللعب بالـ sliders** ← النتيجة تتحدث فورياً + ألوان توضيحية
6. **يقارن بين المجموعات** ← جدول يظهر الفروقات حسب المادة/المنطقة/...
7. **يحفظ السيناريو أو يصدّر النتائج** ← لاستخدامه لاحقاً أو مشاركته

## 🚀 خطة التنفيذ المقترحة (Phased Rollout)

### المرحلة 1: MVP (أسبوع 1-2)
- [ ] تدريب نموذج انحدار خطي على داتا واحدة
- [ ] Sliders تفاعلية + تنبؤ فوري
- [ ] منع أساسي لـ Data Leakage (استبعاد الأعمدة ذات r > 0.85)
- [ ] عرض R² و MAE

### المرحلة 2: تحسينات الجودة (أسبوع 3)
- [ ] دعم المجموعات (Group Comparison)
- [ ] فواصل الثقة (Confidence Intervals)
- [ ] معالجة الحالات الحافة (<10 عينات، قيم خارج النطاق)
- [ ] حفظ تفضيلات المستخدم

### المرحلة 3: التعلم التراكمي (أسبوع 4)
- [ ] دمج بيانات متعددة المشاريع
- [ ] خريطة توحيد الأعمدة (Column Mapping UI)
- [ ] Cache للنماذج المدربة لتسريع التحميل
- [ ] زر "إعادة التدريب" عند إضافة داتا جديدة

### المرحلة 4: ميزات متقدمة (اختياري)
- [ ] دعم نماذج غير خطية (Random Forest, XGBoost)
- [ ] تصدير النموذج كـ API
- [ ] مشاركة السيناريوهات عبر رابط
- [ ] توصيات تلقائية: "لرفع النتيجة 10 نقاط، زد ساعات الدراسة 1.5 ساعة"
