### **The Master Prompt: Implementing Dual-Mode Logic in Axiom**

> **Objective:** > Implement the logic and UI differentiation between **Guided Mode** and **Expert Mode** within the project workspace, ensuring each mode serves its specific user persona while maintaining full data transparency.
>
> ---
>
> ### **1. Guided Mode Logic (The AI Partner)**
> **Persona:** Non-specialist / Business User.
> **Operational UI:**
> - **Language:** Use natural, outcome-based language (e.g., "Grow," "Predict," "Find Patterns").
> - **Tools:** Surface simplified 'Action Chips' in the chat (Auto-clean, Forecast, Segment Customers).
> - **Visuals:** Auto-select the best chart type based on the data.
> - **Behind the Scenes:** Perform a 'Smart Scan' of the data to hide complexity but provide a "View Technical Details" toggle for transparency.
>
> ### **2. Expert Mode Logic (The Data Scientist)**
> **Persona:** Data Analyst / Specialist.
> **Operational UI:**
> - **Language:** Use technical, tool-based language (e.g., "One-Hot Encoding," "Hyperparameters," "Random Forest").
> - **Tools:** Reveal the 'Applied Steps' editor and the full 9-section sidebar (DATA, ANALYSIS, INSIGHTS).
> - **Manual Control:** Allow manual selection of algorithms, cleaning methods, and chart axes.
> - **Interactivity:** The chat side-panel acts as a 'Command Line' for the dashboard.
>
> ---
>
> ### **3. The "Context Switcher" Intelligence**
> **Decision Engine:**
> - If the user is in **Guided Mode** and asks a technical question (e.g., "What is the R-squared?"), answer clearly but provide a link: *"Switch to Expert Mode for full statistical breakdown."*
> - If the user is in **Expert Mode** and asks a broad question (e.g., "Is my business healthy?"), provide a high-level summary but keep all technical panels visible for deep-diving.
>
> ### **4. Implementation Task:**
> 1. Create a `modeContext.tsx` to handle the state between `Guided` and `Expert`.
> 2. Update the `ChatPanel` to render different 'Suggested Chips' based on the active mode.
> 3. Implement the **"Technical Transparency"** feature: any simplified result in Guided Mode should have an underlying JSON/Code block that is fully editable when switched to Expert Mode.
> 4. Ensure the **Data Context Bar** (Task #159) stays visible in both modes but shows different metadata (Simple stats for Guided, Full schema for Expert).
