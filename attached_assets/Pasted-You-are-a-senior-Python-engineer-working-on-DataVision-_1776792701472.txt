**auto_discovery.py**
- Function: run_discovery(df_cleaned, business_context) → list of Finding
- Finding is a dataclass:
    @dataclass
    class Finding:
        type: str  # "correlation"|"seasonality"|"outlier"|"quality"|"kpi"
        title: str
        description: str  # max 120 chars
        columns: list
        severity: str  # "info"|"warning"|"critical"
        value: float = None  # the numeric signal (e.g. correlation r=0.87)

- Correlation scan: Pearson r for all numeric pairs, flag pairs where 
    abs(r) > 0.65. Title: "Strong link: {col_a} vs {col_b} (r={r:.2f})"
    
- Seasonality scan: only if a datetime column exists AND a numeric column 
    has > 52 non-null rows. Use pandas resample("ME").mean() and compute 
    coefficient of variation across months. If CV > 0.15 flag as seasonal.
    Title: "{col} shows monthly seasonal pattern (CV={cv:.0%})"
    
- KPI match: if business_context is set, check if any column name 
    contains one of the KPI strings from the profile. Flag as "kpi" type 
    with severity "info". Title: "KPI column detected: {col}"

- Quality flags: if any column has > 20% missing values → "critical"
    If duplicate rows > 1% → "warning"

- Render in app.py Overview section:
    Below the existing KPI metrics, add a "Discoveries" expander.
    Inside: render each Finding as a styled st.container with:
    - Colored left border by severity (teal=info, amber=warning, red=critical)
    - Title in bold, description below, affected columns as st.code pills

- AnomalyAlert dataclass:
    @dataclass
    class AnomalyAlert:
        column: str
        alert_type: str  # "spike"|"drop"|"missing_surge"|"new_outlier"
        severity: str    # "warning"|"critical"  
        message: str     # human-readable, max 160 chars (SMS-safe)

        threshold: float
        
- Detection logic:
    - For each numeric column: if latest value > mean + 2.5*std → "spike"

    - If missing % in any column increased by > 10pp vs previous upload → 
      "missing_surge"
    - KPI columns (matched from business_context) use tighter threshold: 
      1.5*std instead of 2.5*std
